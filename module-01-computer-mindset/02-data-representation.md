## 💡 核心概念

### 为什么是二进制？

> **计算机使用二进制（0 和 1），因为硬件只能区分两种状态**

### 类比：开关

```
想象一个开关：
  - 开 = 1
  - 关 = 0

一个开关能表示 2 种状态 (0 或 1)
两个开关能表示 4 种状态 (00, 01, 10, 11)
三个开关能表示 8 种状态 (000~111)
...

n 个开关能表示 2^n 种状态
```

### 物理实现

| 存储类型 | 0 表示 | 1 表示 |
|----------|--------|--------|
| 电路 | 低电压 | 高电压 |
| 磁盘 | 磁极向下 | 磁极向上 |
| 光盘 | 凹坑 | 平面 |
| SSD | 电子少 | 电子多 |

**关键：** 无论物理实现如何，逻辑上都是 0 和 1。

## 📐 基本概念

### 比特（Bit）

> **比特是信息的最小单位，表示一个二进制位（0 或 1）**

- 英文：**Bit** (Binary Digit)
- 一个比特 = 一个 0 或 1

### 字节（Byte）

> **字节是计算机的基本存储单位，由 8 个比特组成**

```
1 Byte = 8 Bits

一个字节可以表示 256 (2^8) 种不同的值
范围：00000000 ~ 11111111 (二进制)
     0 ~ 255 (十进制)
```

### 存储单位

| 单位 | 大小 | 大约能存储 |
|------|------|-----------|
| 1 Byte (B) | 8 bits | 一个英文字母 |
| 1 Kilobyte (KB) | 1024 B | 一小段文字 |
| 1 Megabyte (MB) | 1024 KB | 一首 MP3 歌曲 |
| 1 Gigabyte (GB) | 1024 MB | 一部电影 |
| 1 Terabyte (TB) | 1024 GB | 数百部电影 |

> 💡 为什么是 1024 而不是 1000？因为 2^10 = 1024，接近 1000

## 🔢 数字如何存储？

### 整数

直接用二进制表示：

```
十进制 → 二进制

5  → 00000101
10 → 00001010
255→ 11111111
```

### 负数

需要特殊编码（补码），但你不需要深究细节：

```
-5 在计算机中存储为：11111011（8 位补码）

关键点：计算机有办法表示负数，你只需要知道"可以表示"
```

### 小数（浮点数）

使用科学计数法的二进制版本：

```
3.14 ≈ 用特殊格式存储（IEEE 754 标准）

你只需要知道：
- 小数也可以存储
- 可能有精度损失（0.1 + 0.2 ≠ 0.3 在计算机中）
```

## 🔤 文本如何存储？

### 编码的概念

> **编码 = 字符和数字之间的映射规则**

```
字符  →  数字  →  二进制

"A"  →  65   →  01000001
"B"  →  66   →  01000010
"中" →  20013 → 0100111000101101
```

### 常见编码标准

| 编码 | 说明 | 特点 |
|------|------|------|
| **ASCII** | 最早的标准 | 只能表示英文和符号（128 个字符） |
| **GB2312/GBK** | 中文编码 | 能表示汉字，但不同语言不兼容 |
| **UTF-8** | 现代标准 | 能表示所有语言，向后兼容 ASCII |

### UTF-8 编码示例

```python
# Python 示例（你不需要会写，能读懂即可）

text = "Hello 你好"

# 查看每个字符的 Unicode 编码
for char in text:
    print(f"{char}: {ord(char)}")

# 输出：
# H: 72
# e: 101
# l: 108
# l: 108
# o: 111
#  : 32
# 你：20320
# 好：22909
```

## 🖼️ 图片如何存储？

### 位图（Bitmap）

图片被分成许多小格子（像素），每个像素存储颜色：

```
一张 10x10 的图片 = 100 个像素

每个像素存储颜色信息：
- 黑白图片：1 bit (0=黑，1=白)
- 灰度图片：8 bits (0-255，256 级灰度)
- 彩色图片：24 bits (红 8 位 + 绿 8 位 + 蓝 8 位)

10x10 的彩色图片 = 100 像素 × 24 bits = 2400 bits = 300 Bytes
```

### RGB 颜色模型

```
每种颜色由红 (R)、绿 (G)、蓝 (B) 组合而成：

红色：(255, 0, 0)
绿色：(0, 255, 0)
蓝色：(0, 0, 255)
白色：(255, 255, 255)
黑色：(0, 0, 0)

每个值范围：0-255（一个字节）
```

### 示例：一个简单的图片

```
2x2 的图片：
┌─────┬─────┐
│ 红  │ 绿  │
├─────┼─────┤
│ 蓝  │ 白  │
└─────┴─────┘

在计算机中存储为：
[(255,0,0), (0,255,0), (0,0,255), (255,255,255)]

共 12 个字节
```

## 🎵 其他数据类型

### 音频

```
声音 → 波形 → 数字采样 → 存储

CD 音质：
- 采样率：44100 次/秒
- 位深度：16 bits
- 声道：2（立体声）

1 秒音频 = 44100 × 16 × 2 = 1,411,200 bits ≈ 176 KB
```

### 视频

```
视频 = 一连串图片（帧）+ 音频

30 帧/秒的视频：
1 秒 = 30 张图片 + 1 秒音频

所以视频文件很大！
```

## 🔍 深入思考

### 为什么理解数据表示很重要？

1. **理解文件大小**
   - 为什么图片比文字大？
   - 为什么视频比音频大？

2. **理解编码问题**
   - 为什么有时看到乱码？
   - 为什么 UTF-8 是推荐编码？

3. **理解 AI 的输入**
   - AI 处理的文字最终都是数字
   - Token 就是一种数字表示

4. **做出合理决策**
   - 选择什么格式存储数据
   - 理解存储和带宽需求

### 常见误解

| 误解 | 正确理解 |
|------|----------|
| "计算机懂文字" | 计算机只懂数字，文字是编码后的数字 |
| "1GB=1000MB" | 计算机中 1GB=1024MB（但硬盘厂商用 1000） |
| "数字越大精度越高" | 浮点数有精度限制，不是所有小数都能精确表示 |

## ✅ 本课小结

```
核心要点：

1. 计算机只能处理数字，所有信息都要转换成数字

2. 二进制（0 和 1）是因为硬件只能区分两种状态

3. 比特是最小单位，字节（8 比特）是基本单位

4. 文本通过编码（如 UTF-8）转换成数字

5. 图片是像素的颜色值数组

6. 理解数据表示帮助你：
   - 理解文件大小
   - 解决编码问题
   - 理解 AI 如何处理信息
```
