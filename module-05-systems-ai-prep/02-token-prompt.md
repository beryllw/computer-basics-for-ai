## 💭 从一个问题开始

**AI 如何"理解"文本？**

AI 看到的不是文字，而是数字：

```
"Hello World"
   ↓ 分词
["Hello", " World"]
   ↓ 编码
[15496, 2159]
   ↓ AI 处理
数字运算
   ↓ 解码
下一个词预测
```

这就是 **Token** 的作用。

---

## 💡 核心概念

### 什么是 Token？

> **Token = AI 处理文本的基本单位**

**Token 可以是**：
- 一个单词：`"cat"` → 1 个 token
- 一个词的一部分：`"learning"` → `["learn", "ing"]` → 2 个 tokens
- 一个标点：`"."` → 1 个 token
- 一个空格：`" "` → 1 个 token

### Token 数量估算

```
英文：1 token ≈ 4 个字符 ≈ 0.75 个单词
中文：1 个汉字 ≈ 1-2 个 tokens

例子：
"Hello, World!" → 约 3-4 tokens
"你好，世界！" → 约 5-6 tokens
```

### 为什么 Token 重要？

**1. 影响费用**
```
API 计费通常按 token 计算
输入 token + 输出 token = 总费用
```

**2. 影响速度**
```
Token 越多，处理越慢
```

**3. 有限制**
```
上下文窗口限制：
- Claude: 200K tokens
- GPT-4: 128K tokens
- 超出限制会被截断
```

## 📝 什么是 Prompt？

### Prompt 的定义

> **Prompt = 给 AI 的输入指令**

告诉 AI 你想要什么。

### Prompt 的结构

```
系统指令（可选）
   ↓
用户指令
   ↓
上下文/示例（可选）
   ↓
AI 响应
```

### 好 Prompt vs 差 Prompt

| 差 Prompt | 好 Prompt |
|-----------|-----------|
| "解释这个" | "用初学者能理解的语言解释什么是递归" |
| "写代码" | "用 Python 写一个函数，计算列表的平均值" |
| "帮我改改" | "请检查这段代码的语法错误，并解释如何修复" |

---

## 🎨 Prompt 技巧

### 技巧 1：明确具体

```
❌ 模糊："写点关于 AI 的东西"
✅ 具体："用 300 字介绍 AI 在医疗领域的应用"
```

### 技巧 2：提供上下文

```
❌ 没有上下文："这个对吗？"
✅ 有上下文："我刚学了变量，我的理解是变量像带标签的盒子。这个理解对吗？"
```

### 技巧 3：指定格式

```
❌ 不指定："给我一些建议"
✅ 指定："给我 5 条学习建议，用列表格式，每条包含原因"
```

### 技巧 4：提供示例

```
请按照以下格式总结文章：

标题：[文章标题]
要点：
- [要点 1]
- [要点 2]
- [要点 3]

文章：[粘贴文章内容]
```

### 技巧 5：分步骤

```
请分步骤完成以下任务：

1. 先分析这段代码的功能
2. 指出可能的 bug
3. 提供修复建议
4. 解释修复的原因

代码：[粘贴代码]
```

## 🔍 深入理解

### Token 是如何计算的？

**分词过程（Tokenization）**：

```
文本：I'm learning AI

1. 规范化：I'm learning AI
2. 分词：["I", "'", "m", " learn", "ing", " AI"]
3. 编码：[40, 34, 227, 1234, 567, 890]
4. Token 数：6
```

**中文例子**：

```
文本：我正在学习人工智能

分词：["我", "正在", "学习", "人工", "智能"]
Token 数：5-8（取决于具体分词方式）
```

### 上下文窗口

**什么是上下文窗口**？

> AI 一次性能处理的 token 数量限制

**包括**：
- 你的 Prompt
- AI 已经生成的内容
- 系统指令

**超出限制会怎样**？

```
最早的内容会被"遗忘"

对话历史：
[消息 1] ← 超出限制，被截断
[消息 2]
[消息 3]
...
[消息 N] ← AI 只能看到这些
```

### 温度和随机性

**Temperature 参数**：

```
低温度（0.1-0.3）：
- 输出更确定
- 适合事实性问题
- 结果更可预测

高温度（0.7-1.0）：
- 输出更多样
- 适合创意写作
- 结果更随机
```

## 🧪 实践示例

### 示例 1：简单问答

```
Prompt:
"什么是 Python？"

AI Response:
"Python 是一种高级编程语言..."
```

### 示例 2：代码生成

```
Prompt:
"用 Python 写一个函数，接收一个数字列表，返回平均值。
要求：
1. 处理空列表的情况
2. 添加文档字符串
3. 包含使用示例"

AI Response:
```python
def calculate_average(numbers):
    """计算数字列表的平均值"""
    if not numbers:
        return 0
    return sum(numbers) / len(numbers)

# 使用示例
print(calculate_average([1, 2, 3, 4, 5]))  # 输出 3.0
```
```

## 💡 高级技巧

### Chain of Thought（思维链）

```
Prompt:
"请逐步思考这个问题：

小明有 5 个苹果，他给了小红 2 个，
然后又买了 3 个。现在他有几个苹果？

让我们一步一步思考：
1. 小明开始有 5 个苹果
2. 给小红 2 个后：5 - 2 = 3 个
3. 又买 3 个后：3 + 3 = 6 个
4. 所以答案是 6 个
```

### Few-Shot Prompting（少样本提示）

```
Prompt:
"将下列中文翻译成英文：

例子 1:
输入：你好
输出：Hello

例子 2:
输入：谢谢
输出：Thank you

现在翻译：
输入：再见
输出：___"
```

### 系统指令

```
系统指令：
你是一位专业的代码审查员。
你的任务是：
1. 检查代码错误
2. 提出改进建议
3. 解释原因

用户输入：
[代码]
```

## 📝 关键术语

| 术语 | 英文 | 解释 |
|------|------|------|
| Token | Token | AI 处理文本的单位 |
| Prompt | Prompt | 给 AI 的指令 |
| 分词 | Tokenization | 将文本分成 token |
| 上下文 | Context | AI 能看到的内容 |
| 上下文窗口 | Context Window | AI 能处理的 token 上限 |
| 温度 | Temperature | 控制输出的随机性 |
| 系统指令 | System Prompt | 设定 AI 的角色 |

## ✅ 本课小结

```
核心要点：

1. Token：
   - AI 处理文本的单位
   - 影响费用和速度
   - 有数量限制

2. Prompt：
   - 给 AI 的指令
   - 要具体、明确
   - 提供上下文

3. 好 Prompt 技巧：
   - 明确具体
   - 提供上下文
   - 指定格式
   - 分步骤

4. 理解限制：
   - 上下文窗口有限
   - 超出会被截断
   - 合理安排内容
```
